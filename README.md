## Reflection

Artemis Financial is a financial services company that wanted to improve the security of one of its software applications. The company needed help identifying security vulnerabilities and strengthening the application so it could better protect sensitive customer and financial information.

One thing I did well was learning how to look at vulnerabilities as real risks instead of just errors in a report. Secure coding is important because software can function correctly and still be unsafe if security is not considered. Strong software security helps protect customer data, reduces company risk and supports trust in the business.

The vulnerability assessment was challenging because I had to understand what the dependency-check results meant and decide which issues actually mattered. It was helpful because it showed me how security tools can support development instead of being separate from it.

I increased layers of security by reviewing dependency vulnerabilities, considering secure coding practices, and recommending mitigation steps such as updating dependencies and using secure communication practices. In the future, I would use tools like OWASP Dependency-Check, static analysis tools, secure coding standards, and vulnerability documentation to assess risks and decide which mitigation techniques make the most sense.

To make sure the code and application were functional and secure, I rebuilt and tested the application after making changes. After refactoring, I ran the dependency check again to confirm whether vulnerabilities were reduced and to make sure I had not introduced new issues.

The resources and tools I used included Java, Maven, OWASP Dependency-Check, vulnerability reports, and secure coding guidelines. These will be useful in future assignments and real projects because security should be part of the development process from the beginning.

From this assignment, I could show future employers my vulnerability assessment report, my ability to interpret security findings, and my understanding of secure software development.
